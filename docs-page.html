What is doozer?

.

Target audience

.

Feature set

  • Simple, action based, script langauge.
  • Screenshots comparison vs goldens.
  • Semantic Locators support.
  • UI elements assertions.
  • One test script - multiple browsers.
  • Parallel execution.
  • ...
Detailed logging with failures debugging helpers (window hierarchy + screenshot at fail) Web based image comparison approvals

Doozer future

  • Support for the following platforms:
    • Android (based on UiAutomator)
    • iOS (based on XCUITest)
    • Windows (based on WinAppDriver)
  • Support for a11y, l10n and i18n verification.
  • Platform specific UI element locator.
  • VS Code code completion.
  • VS Code debugger.
  • VS Code action recorded.
  • Extend SemanticLocators with Android and iOS implementation.
Order of the implementation is not decided yet and will depend on the interest shown by the users. Is one of them important to you? Feel free to drop us an email at: info [to be changed by a script] softtest.dev

Installation

.

Prerequisites

Mandatory:

  • Java SDK
  • Maven

Optional:

  • Visual Studio Code
  • VSCode: Doozer extension

Maven dependency

.

								
	<dependency>
		<groupId>dev.softtest</groupId>
		<artifactId>doozer</artifactId>
		<version>0.2.3</version>
	</dependency>

Get Started

.

Project setup

  • Setup Maven project
  • Add doozer dependecy into pom.xml
  • Create a base test class

Writing first script

  • Create new test script
  • Add first actions

								
	url args:”http://doozer.softtest.dev/firstscript”
	type selector:”SL({textbox ‘name’}) args:”Doozer”
	click selector:”SL({button ‘submit’})
	assertInnerTest selector:”By.classname(‘greeting’)” args:”Hello Doozer!”
